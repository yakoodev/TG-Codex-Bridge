# G1 — `/new`: новый Codex чат + новый Topic

## Цель
Создавать новый topic для новой codex-сессии в той же директории.

## Сделать
- Команда `/new` в topic:
  - создать новый Telegram topic в группе
  - создать новую запись `topics` (project_id тот же)
  - в новом topic запустить `codex` и инициировать новую сессию:
    - отправить в stdin `/new\n` (или другой триггер, если codex CLI так делает; уточнить реальным выводом)
- Попытаться распарсить новый `codex_chat_id` из вывода (см. G2).

## Критерии приёмки
- Старый topic остаётся работоспособным со своим контекстом.
- Новый topic стартует с чистой историей.

## Как проверить
- В старом topic “запомни X”, затем `/new`, в новом спросить “что я просил?” → не должен знать.
