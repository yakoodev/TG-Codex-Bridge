# C1 — Telegram: роутинг апдейтов + фильтр по пользователю

## Цель
Правильно разделять личку и топики, и обслуживать только ALLOWED_USER_ID.

## Сделать
- Подключить `Telegram.Bot` (long polling или webhook — выбрать long polling для простоты).
- На каждый апдейт:
  - если `from.id != ALLOWED_USER_ID` → игнор без побочек
- Роутинг:
  - Личка: команды `/mkproj`, `/help`
  - Группа topic: текст → job; команды `/status` `/cancel` `/new`
- Логи: тип апдейта + chat/thread id.

## Критерии приёмки
- Чужие пользователи не могут ни создать topic, ни триггернуть codex.
- В личке `/mkproj` отвечает.
- В топике текст запускает job.

## Как проверить
- Написать боту с другого аккаунта → 0 реакции.
- С разрешённого → `/mkproj` работает.
